1. 스키마에 대해서 설명
	DBMS 특성과 구현 환경을 감안한 데이터 구조

2. ddl명령어에 대해 설명
 1. create 
	테이터베이스 오브젝트 생성
 2. alter
	테이터베이스 오브젝트 변경
 3. drop
	테이터베이스 오브젝트 삭제
 4. truncate
	테이터베이스 오브젝트 내용 삭제

3. dml명령어에 대해 설명
 1. insert
	삽입 형태로 신규 데이터를 테이블에 저장
 2. select
	테이블의 내용을 조회
 3. update
	테이블의 내용을 변경
 4. delete
	테이블의 내용을 삭제

4. 트랜잭션에 대해서 설명
	동시에 다수의 작업을 독립적으로 안전하게 처리하기 위한 상호 작용 단위

5. dcl명령어에 대해 설명
 1. grant
	데이터베이스 사용자 권한 부여
 2. revoke
	데이터베이스 사용자 권한 회수

6. tcl명령어에 대해 설명
 1. commit
	트랜잭션 확정
 2. rollback
	트랜잭션 취소
 3. savepoint
	복귀지점 설정

7  데이터 사전에 대해서설명
	데이터 베이스의 데이터를 제외한 모든 정보가 있다. 데이터 사전의 내용을 변경하는 권한은 시스템이 가지며, 사용자에게는 읽기 전용 테이블 형태로 제공되므로 단순 조회만 가능하다

8. 데이터베이스의 정의 4가지
	통합 데이터 - 중복이 최소화된 데이터
	저장 데이터 - 컴퓨터를 통해 접근 가능한 저장 매체에 저장된 데이터
	운영 데이터 - 조직의 목적을 위해 존재가치가 확실하고 반드시 필요한 데이터
	공유 데이터 - 여러 응용 프로그램이 공동으로 사용하는 데이터

9. 인덱스에 대해 설명
	검색을 빠르게 하기 위한 데이터 구조

10. 뷰에 대해 설명
	하나 이상의 물리 테이블에서 유도되는 가상의 논리 테이블
